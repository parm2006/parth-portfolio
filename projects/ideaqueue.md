# IdeaQueue

**Status:** Restart planned  
**Repo:** Local only for now

IdeaQueue should become a lightweight idea capture app accessible from both phone and laptop.

## Desired Product

When Parth has a thought in class or away from the laptop, he should be able to quickly add it from his phone. Ideas should be stored in a database, prioritized, and later popped from a heap/priority queue when he wants to review them.

## Current Decision

The current implementation is considered unreliable. The frontend should be restarted, and the backend may also be restarted if repairing it takes more effort than rebuilding cleanly.

## Useful Old Ideas

- Priority increases over time.
- Sorting by priority and creation time.
- Categories and filters.
- Pop endpoint.
- Small widget view.

Treat those as product ideas, not code that must be preserved.
